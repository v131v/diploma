     Keywords:                                           Erasure Coding (EC) is a security measure that eliminates the need for complete data replication by
     Erasure Coding                                      allowing data to be reconstructed from parity pieces. This method offers increased data redundancy,
     OpenStack Swift                                     efficiency, lowered storage costs, and enhanced fault tolerance making it a preferred alternative to
     Reed-Solomon Codes                                  replication in Object Storage Systems. The fundamental concept behind EC is to encrypt a specific




                                                        tp
     Cloud Storage System                                amount of data in a manner that guarantees the transfer of all coded pieces without any loss. As
                                                         data volumes continue to rapidly increase, the time efficiency of the EC method becomes crucial in
                                                         ensuring optimal system performance. Various variables, including the algorithm employed, data size,
                                                         number of storage nodes, hardware resources, and network conditions, can influence the speed of EC
                                                         operations. Hence, we inspect the Erasure Coding algorithm known as Reed-Solomon Codes (RS). We




                                                          ee
                                                         explore the algorithm’s encoding speed, taking into account factors such as data size and the generation
                                                         of parity blocks. By evaluating and enhancing existing mechanisms, our research aims to address
                                                         time efficiency and fault tolerance challenges in cloud-based object storage systems. To enhance data
                                                         retrieval and storage processes, we comprehensively examine time efficiency mechanisms, such as
                                                         data placement policies and scheduling algorithms. Additionally, we investigate the time taken for




                                                            rr
                                                         an open-source cloud storage system, OpenStack Swift, to store data by analyzing two datasets and
                                                         determining the duration required to store these files. Furthermore, our study assesses fault tolerance
                                                         mechanisms, including redundancy schemes, error correction codes, and distributed data placement
                                                         strategies, to bolster system resilience. We propose different approaches to minimize access latency,
                                                         improve overall time efficiency, and ensure data availability, even in the presence of failures.
